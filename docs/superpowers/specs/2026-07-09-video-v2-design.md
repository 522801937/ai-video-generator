# AI 科普视频生成器 V2 — 设计文档

> **状态**: 待用户审阅
> **日期**: 2026-07-09
> **目标**: 将 karpathy-video 从"文字+照片幻灯片"升级为"精美动画信息图+概念插画"的一键视频生成桌面软件

---

## 1. 问题诊断

V1 核心问题：
- 幻灯片 = 爬取的背景图 + 暗色遮罩 + 白色文字 → 本质是自动配图 PPT 转视频
- 动画仅有 Ken Burns 慢速缩放 + 淡入淡出转场
- 无 GUI，命令行操作
- 图片与内容语义关联度低

## 2. 产品目标

用户可以：写好文案 → 输入软件 → 自动生成画面精美、有动画效果、有概念图/信息图视觉辅助的科普视频，可选字幕和配音，一键导出 MP4。

定位：类似 Animated PPT + 科普短视频的混合体，效果对标高质量知识类视频（如 Kurzgesagt、3Blue1Brown 的风格参考）。

## 3. 技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 动画渲染引擎 | HTML5 Canvas + Playwright 帧捕获 | 编辑器预览 = 最终效果（WYSIWYG）、前端生态丰富 |
| 桌面打包 | PyInstaller + 本地 Web 服务 | Python 后端的生态 + 浏览器前端的表现力 |
| 图片来源 | 爬取 → 纯动画兜底 → API 可选 | 免费优先、离线可用、灵活升级 |
| 配音 | Edge TTS（免费微软神经网络） | V1 已集成、质量足够、无需 API |
| 字幕 | SRT 生成 + FFmpeg 烧录 | V1 已实现，直接复用 |

## 4. 整体架构

```
桌面壳 (.exe)
├── Python 后端 (FastAPI, localhost:8765)
│   ├── 图片爬取/API 管理
│   ├── TTS 配音引擎 (Edge TTS)
│   ├── 视频合成调度 (FFmpeg + Playwright)
│   └── 静态文件托管 (前端打包产物)
│
├── 浏览器前端 (React)
│   ├── 文案编辑器
│   ├── 场景编排器 (选择/配置场景类型)
│   ├── 实时动画预览 (HTML5 Canvas)
│   └── 导出控制面板
│
└── 动画渲染引擎 (HTML5)
    ├── 预览模式: 浏览器实时渲染
    └── 导出模式: Playwright 逐帧截图 → FFmpeg 合成 MP4
```

## 5. 场景/动画系统

### 5.1 10 种场景模板

| 场景类型 | 用途 | 动画特点 |
|----------|------|----------|
| Title | 封面标题 | 大标题弹性弹出、背景渐变、粒子装饰 |
| Concept | 概念引入 | 逐字打字、图标弹出、SVG 插画 |
| Bullets | 要点列举 | 列表逐项滑入、高亮强调 |
| Compare | 对比分析 | 双栏展开、差异动画标注 |
| Flow | 流程图 | 节点逐个出现、连线绘制动画 |
| Chart | 数据图表 | 柱状图生长、数字滚动、饼图展开 |
| Quote | 金句强调 | 引号放大、文字居中淡入 |
| Image | 图片展示 | 全屏图片 + 叠加文字、Parallax |
| Timeline | 时间线 | 事件节点从下至上出现 |
| Outro | 片尾 | 总结语 + 二维码/Logo |

### 5.2 动画原语

| 原语 | 效果 |
|------|------|
| `fadeIn` | 元素淡入 |
| `slideIn` | 从侧边/下方滑入 |
| `scaleIn` | 弹性放大 |
| `typewriter` | 逐字打出 |
| `counterUp` | 数字滚动 |
| `drawLine` | 线条渐绘 |
| `staggerList` | 列表逐项动画 |
| `floatParticles` | 背景浮动粒子 |

### 5.3 场景定义格式

每个场景是一个 JSON 对象 + 一个 HTML 组件：

```json
{
  "id": "scene_01",
  "type": "title",
  "duration": 4.0,
  "data": {
    "title": "什么是 AI Agent？",
    "subtitle": "从大模型到自主智能体",
    "icon": "robot-outline"
  },
  "animation": {
    "bgGradient": { "from": "#0a0a2e", "to": "#1a1a4e" },
    "particles": true
  }
}
```

前端读取 JSON → 渲染对应的 React 场景组件 → 动画按时间线播放。

## 6. 图片流水线

```
关键词 → Bing/Unsplash 爬取
              │
    有结果 ───┤          ┌→ 缓存为本地图片 → 用于场景
              │          │
    无结果 ───┤→ API? ───┤
              │    (可选) │
              │          └→ DALL·E / 通义万相 生成
              │
              └→ 纯动画兜底 (抽象几何背景 + SVG 图标)
```

默认"免费模式"开箱即用。用户可在设置中填入 API Key 启用 AI 生图。

## 7. 桌面打包

| 组件 | 方案 |
|------|------|
| Python 后端 + 依赖 | PyInstaller 打包 (~200MB) |
| 前端 | Vite 构建为静态文件，FastAPI 托管 (~3MB) |
| FFmpeg | ffmpeg.exe 打入 (~80MB) |
| Chromium | 复用 playwright 的 chromium (~150MB) |
| Edge TTS | 系统自带，不打包 |
| **预估总包大小** | **~430MB** |

启动流程：双击 exe → 启动后端 → 打开浏览器 → 进入编辑器。

## 9. 与 V1 的关系

V1 代码保留不动，V2 作为新模块开发：
- 复用：`tts_engine.py`（TTS）、`subtitle_generator.py`（字幕）、`media_fetcher.py`（爬图）
- 重写：`slide_generator.py`（不再用 Pillow 渲染静态图）、`video_composer.py`（从 MoviePy 升级为 Playwright + FFmpeg）
- 新增：`server/`（FastAPI 后端）、`web/`（React 前端）、`renderer/`（动画引擎）、`scenes/`（场景组件）

## 10. 遗留决策

以下功能先不做，后续迭代考虑：
- 配音：先复用 Edge TTS，不接入云端 TTS
- 本地小模型：先不做，图片走爬虫+动画兜底
- 多人协作 / 云端同步
- 视频模板市场
